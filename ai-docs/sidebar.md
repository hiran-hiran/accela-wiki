# サイドバー表示ロジック

## 表示内容の決定

サイドバーには`data/`ディレクトリの構造が自動的に反映されます。

### 表示名の決定順序

1. `index.md`のFront Matter `title`
2. `index.md`の最初の`# 見出し`
3. ディレクトリ名（数字プレフィックス除去後）

### 例

```
data/02_guide/
├── index.md  ← title: "使い方ガイド"
└── ...
```

→ サイドバーに「使い方ガイド」と表示

```
data/03_api/
├── 01_endpoints.md
└── 02_errors.md
（index.mdなし）
```

→ サイドバーに「api」と表示（ディレクトリ名から）

## リンクの有無

| 条件 | サイドバー表示 |
|------|---------------|
| `index.md`あり + 本文あり | リンクあり |
| `index.md`あり + 本文なし（titleのみ） | リンクなし |
| `index.md`なし | リンクなし |

### 本文なしの判定

Front Matterの後に空白・改行以外のコンテンツがない場合、「本文なし」と判定されます。

```markdown
---
title: グループ名
---
```
↑ 本文なし（リンクなし）

```markdown
---
title: セクション名
---

# セクション名

説明文...
```
↑ 本文あり（リンクあり）

## 並び順

数字プレフィックスの昇順でソートされます。

```
data/
├── 01_first.md    ← 1番目
├── 02_second.md   ← 2番目
├── 10_tenth.md    ← 3番目（10 > 2）
└── 03_third.md    ← 4番目として表示されるべきだが...
```

**注意**: 連番を飛ばすと意図しない順序になる可能性があります。
常に連番（01, 02, 03...）を維持してください。

## 階層表示

サイドバーは再帰的にネストを表示します。

```
├── セクション1
│   ├── ページA
│   └── サブセクション
│       ├── ページB
│       └── ページC
└── セクション2
```

深い階層はインデントで表現されます。

## currentクラス

現在表示中のページに対応するサイドバーリンクには`current`クラスが付与されます。
これによりCSSで現在位置をハイライト表示できます。
