# 禁止事項・推奨パターン

## 禁止事項

### 編集禁止のディレクトリ

| ディレクトリ | 理由 |
|-------------|------|
| `app/` | フレームワーク本体。変更するとシステムが壊れる |
| `vendor/` | Composerの依存パッケージ |

### ファイル操作

- `env.php`に機密情報（パスワード、APIキー等）を書かない
- `data/`以外にマークダウンファイルを配置しない
- `.htaccess`を削除・変更しない（ルーティングに必要）

### 命名

| 禁止 | 理由 |
|------|------|
| スペースを含むファイル名 | URLエンコードが必要になる |
| 日本語ファイル名 | 環境依存の問題が発生しやすい |
| 大文字を含むファイル名 | OS間の互換性問題 |
| `_`で始まるファイル名 | 将来的に予約される可能性 |

## 非推奨パターン

### ファイル命名

```
❌ Getting Started.md      → スペース禁止
❌ GettingStarted.md       → キャメルケース非推奨
❌ getting_started.md      → スネークケース非推奨
✅ getting-started.md      → ケバブケース推奨
✅ 01_getting-started.md   → プレフィックス + ケバブケース
```

### ディレクトリ構造

```
❌ 5階層以上のネスト
❌ プレフィックスの欠番（01, 02, 05...）
❌ 同じディレクトリに100以上のファイル

✅ 3階層まで推奨
✅ 連番のプレフィックス（01, 02, 03...）
✅ 適度なグルーピング
```

### Front Matter

```
❌ titleを省略（ファイル名がタイトルになる）
❌ YAMLの文法エラー

✅ 常にtitleを明記
✅ descriptionも記述推奨
```

## 推奨パターン

### ファイル作成時のチェックリスト

1. [ ] ファイル名はケバブケース
2. [ ] 数字プレフィックスで順序指定
3. [ ] Front Matterにtitleを記述
4. [ ] 最初の見出しはtitleと一致
5. [ ] 適切な階層に配置

### ディレクトリ作成の判断基準

| 状況 | 推奨 |
|------|------|
| 関連ページが3つ以上 | ディレクトリにまとめる |
| 単独のページ | ルート直下に配置 |
| サブカテゴリが必要 | ネストしたディレクトリ |

### index.mdを作るべきか？

| 状況 | 推奨 |
|------|------|
| セクションの説明が必要 | index.md作成（本文あり） |
| グループ名だけ変えたい | index.md作成（titleのみ） |
| ディレクトリ名で十分 | index.md不要 |

## トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| サイドバーに表示されない | 拡張子が`.md`でない | ファイル名確認 |
| ページが404 | プレフィックス付きでアクセス | URLからプレフィックス除去 |
| 順番がおかしい | プレフィックスが不連続 | 連番に修正 |
| タイトルがファイル名 | Front Matterがない | title追加 |
| リンクが動作しない | パスが間違っている | 絶対パス（/から始まる）を使用 |

## 出力時の注意

### 複数ファイルを生成する場合

1. ディレクトリ構造を先に示す
2. 各ファイルの内容を順番に出力
3. ファイルパスを明記

```
## 生成するファイル構造

data/
├── index.md
└── 01_guide/
    ├── index.md
    └── 01_basics.md

## data/index.md

\`\`\`markdown
---
title: ...
---
...
\`\`\`

## data/01_guide/index.md

\`\`\`markdown
...
\`\`\`
```
