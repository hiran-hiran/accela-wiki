# AIを使った開発

Accela Wikiは「マークダウンを置くだけ」で動作するため、AIとの相性が抜群です。
ChatGPT、Claude、Cursor等のAIツールにドキュメント生成を任せることで、効率的にWikiを構築できます。

## なぜAIと相性が良いのか

| 従来のツール | Accela Wiki |
|-------------|-------------|
| 設定ファイルの生成が必要 | マークダウンだけでOK |
| ビルドコマンドの実行が必要 | 置くだけ |
| Node.js環境が必要 | PHPサーバにFTP |

AIはマークダウンの生成が得意です。Front Matterも正確に出力できます。

## 基本的なワークフロー

1. AIにドキュメントの内容を指示
2. 生成されたマークダウンを`data/`に配置
3. ブラウザで確認

これだけでWikiが完成します。

## AI向けドキュメント

プロジェクトルートの`ai-docs/`ディレクトリにAI向けの仕様書が含まれています。

```
ai-docs/
├── README.md        - 概要
├── structure.md     - ファイル配置ルール
├── frontmatter.md   - Front Matter仕様
├── routing.md       - URL生成ルール
├── sidebar.md       - サイドバー表示ロジック
├── examples.md      - サンプル
└── guidelines.md    - 禁止事項・推奨パターン
```

AIツールにこれらのファイルを読み込ませることで、Accela Wikiの仕様を理解した上でドキュメントを生成できます。

## 使用例

### Claude Codeの場合

```
ai-docs/を読んで、このプロジェクトのWikiドキュメントを作成して。
以下のセクションが必要:
- インストール方法
- 基本的な使い方
- API リファレンス
```

### Cursorの場合

`ai-docs/`をプロジェクトに含めた状態でCursorを使用すると、コンテキストとして自動的に参照されます。

### ChatGPTの場合

`ai-docs/`の内容をコピー&ペーストで渡すか、ファイルをアップロードしてください。

## プロンプト例

### 社内マニュアルを作成

```
ai-docs/を読んで、以下の社内マニュアルを作成してください。

テーマ: 新入社員向けPCセットアップガイド
セクション:
1. 初期設定
2. 必要なソフトウェアのインストール
3. 社内ネットワークへの接続
4. よくある質問

data/ディレクトリに配置するマークダウンファイルとして出力してください。
```

### APIドキュメントを作成

```
ai-docs/を読んで、以下のAPIのドキュメントを作成してください。

[APIの仕様やコードを貼り付け]

エンドポイントごとにページを分けて、
data/api/ディレクトリに配置するマークダウンファイルとして出力してください。
```

### 既存ドキュメントの整理

```
ai-docs/を読んで、以下のドキュメントをAccela Wiki形式に変換してください。

[既存のドキュメントを貼り付け]

適切な階層構造に整理して、data/ディレクトリに配置するマークダウンファイルとして出力してください。
```
